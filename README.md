# Pricing & Churn Economics Console — прототип

Одностраничный React-дашборд (Vite + Recharts) для Product Owner'а IaaS: крутишь параметры новой тарифной сетки и в реальном времени видишь **прогноз оттока по когортам + дельту MRR/ARPU**, с калькулятором миграции по аккаунту и рычагом точечных скидок.

> **Данные синтетические** — смоделированы по реальному распределению цен/flavor'ов 3HCloud, без реальных payer/счетов/UUID. Вывод симулятора — иллюстрация, не продакшен-модель. В UI постоянный честный бейдж.

## Что внутри

- **Pricing Calculator** — множители цены по тирам + премия за выделенные ядра; калькулятор миграции повайтменно по выбранному аккаунту.
- **Churn Simulator** — композитный `churn_risk` по реальным весам кейса **Price Impact 0.35 · Loyalty 0.25 · Growth 0.20 · Size 0.20**; настраиваемый порог; подсветка top-5%.
- **Economics Dashboard** — KPI до/после **per-currency** (RUB/USD/KZT не смешиваются), 2 графика Recharts, таблица top-at-risk, рычаг точечных скидок (net-MRR с удержанием vs при оттоке).

## Запуск локально

```bash
npm install
npm run dev      # http://localhost:5173
```

## Сборка

```bash
npm run build    # → dist/
npm run preview  # локальный просмотр сборки
```

## Деплой на GitHub Pages

1. Создай репозиторий, например `pricing-churn-console`.
2. В `vite.config.js` поле `base` должно быть `'/<имя-репозитория>/'` (по умолчанию `/pricing-churn-console/`; либо задай через env `GHPAGES_BASE`).
3. Установи деплой-зависимость (уже в `devDependencies`): `gh-pages`.
4. `npm run deploy` — соберёт `dist/` и опубликует в ветку `gh-pages`.
5. В Settings → Pages выбери branch `gh-pages` → ссылка готова для интервью.

```bash
GHPAGES_BASE=/my-repo/ npm run deploy
```

## Сверка формулы оттока (verification)

Логика `src/model.js` зеркалит Python-эталон `verify/churn_ref.py`. Прогнать эталон:

```bash
python3 verify/churn_ref.py
```

`churn_risk` каждого аккаунта в дашборде совпадает с эталоном до 3-го знака на тех же данных и дефолтных множителях (детерминированно, без рандома).

## Ограничения честности (по брифу)

- Никаких вызовов LLM/Claude API в рантайме (промпт-инжиниринг — отдельный кейс, не в продукте).
- Валюты считаются раздельно, не суммируются в один MRR.
- Вывод симулятора не выдаётся за реальные продакшен-числа.
